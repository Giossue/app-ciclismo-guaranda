# Patrones mobile first

Guaranda Go está pensado principalmente para Android, no escritorio.

## Layout

- Usar navegación inferior o patrones similares para secciones principales.
- Priorizar pantallas de una columna.
- Evitar tablas grandes en móvil; usar cards/listas.
- Formularios cortos, agrupados por secciones.
- Acciones críticas siempre visibles durante recorrido.

## Pantallas principales

- Inicio.
- Lista de rutas.
- Mapa.
- Detalle de ruta.
- Recorrido activo.
- Descargas offline.
- Favoritos.
- Perfil.
- Reportar incidencia.
- Chatbot.

## Recorrido activo

Debe priorizar:

- Mapa.
- Estado GPS.
- Tiempo.
- Distancia recorrida/restante.
- Botones pausar/finalizar.
- Reportar incidencia.
- Indicador offline.

## Formularios móviles

- Inputs grandes.
- Labels visibles.
- Validación clara.
- Evitar formularios largos sin división.
- Cámara/adjuntos con feedback inmediato.

## Modo exterior

Diseñar pensando en luz solar, movimiento y poco tiempo de atención:

- Contraste adecuado.
- Tipografía legible.
- Acciones grandes.
- Estados de riesgo muy visibles.
